\doxysubsection{Detailed Description}
An {\ttfamily \mbox{\hyperlink{classjavazoom_1_1jl_1_1player_1_1_audio_device_factory}{Audio\+Device\+Factory}}} class is responsible for creating a specific {\ttfamily \mbox{\hyperlink{interfacejavazoom_1_1jl_1_1player_1_1_audio_device}{Audio\+Device}}} implementation. A factory implementation can be as simple or complex as desired and may support just one implementation or may return several implementations depending upon the execution environment. 

When implementing a factory that provides an \mbox{\hyperlink{interfacejavazoom_1_1jl_1_1player_1_1_audio_device}{Audio\+Device}} that uses class that may not be present, the factory should dynamically link to any specific implementation classes required to instantiate or test the audio implementation. This is so that the application as a whole can run without these classes being present. The audio device implementation, however, will usually statically link to the classes required. (See the Java\+Sound deivce and factory for an example of this.)
